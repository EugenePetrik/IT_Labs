  1. What is 'git stash' and how to use it?

Часто пока вы работаете над одной частью вашего проекта и все находится в беспорядке, у вас возникает желание сменить ветку и поработать над чем-то еще. Сложность при этом заключается в том, что вы не хотите фиксировать наполовину сделанную работу только для того, чтобы иметь возможность вернуться к ней позже. Справиться с ней помогает команда git stash.
Операция stash берет измененное состояние вашей рабочей директории, то есть измененные отслеживаемые файлы и проиндексированные изменения, и сохраняет их в хранилище незавершенных изменений, которые вы можете в любое время применить обратно.

  Добавить файлы в хранилище незавершенных изменений:
$ git stash

  Извлечь файлы из хранилища:
$ git stash pop

Она изменяет указатель HEAD и, опционально, состояние индекса. Также эта команда может изменить файлы в рабочей директории при использовании параметра --hard, что может привести к потере наработок при неправильном использовании.

  Команда 'git reset' выполняет три основных операции:
- Откатиться к конкретному коммиту: $ git reset --hard <commit>
- Откатиться к конкретному коммиту, но файлы на диске остаются без изменений: $ git reset --soft <commit>
- Обновление индекса: $ git reset --mixed <commit>


  3. When we need cherry-pick, and how does it happens?

Команда git cherry-pick используется для того чтобы взять изменения, внесенные каким-либо коммитом, и попытаться применить их заново в виде нового коммита наверху текущей ветки. Это может оказаться полезным чтобы забрать парочку коммитов из другой ветки без полного слияния с той веткой.
  Пример:
1. Для этого нужно перйти на ветку, в которую будем вливать коммит: $ git checkout master
2. Обновить ветку: $ git pull origin master
3. Выполнить команду, указать код коммита: $ git cherry-pick eb042098a5
4. После этого обновить ветку на сервере: $ git push origin master


  4. What does do 'git rm ..' command and what not?

Команда git rm используется в Git для удаления файлов из индекса и рабочей директории.
  Пример:
- Удалить файл: $ git rm text.txt
- Удалить файл из индекса, оставив его при этом в рабочем каталоге: $ git rm --cached text.txt
